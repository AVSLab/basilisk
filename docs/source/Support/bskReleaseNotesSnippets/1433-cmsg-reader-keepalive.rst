- Fixed a use-after-free where a C module's input message (a ``Msg_C`` reader) that subscribed to a stand-alone source could read freed memory if the only Python reference to that source went out of scope and was garbage-collected. The subscribed source is now kept alive for as long as the subscription is live, and released again on ``unsubscribe()``, on re-subscribe, or when the subscriber (or the C module that embeds it) is garbage-collected. This completes the message keep-alive started for the C++ reader direction in issue #676.
